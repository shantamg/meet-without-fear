# Stage 05: Verify

## Inputs

- `../04-implement/output/implementation-notes.md`
- Focused test commands
- Browser or API checks relevant to the changed behavior

## Process

Run the narrowest useful verification first, then broader checks when risk warrants it. Capture failures honestly.

## Outputs

- `output/verification.md`

## Audit

Verification must include exact commands, pass/fail status, and why the checks cover the changed behavior.
