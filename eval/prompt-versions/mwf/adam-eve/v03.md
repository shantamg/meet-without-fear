# MWF Prompt Proposal v03

Scenario: `adam-eve`
Source run: `eval/runs/20260505-164520-adam-eve-iter-01`
Focus: Prevent snake_case feel-heard control-tag leakage.

## Evidence

The run scored `3.5` overall with `mwf_handling` at `3` and a failed hard invariant:

```text
<feel_heard>Y</feel_heard>
```

This appeared in Adam's visible Stage 1 chat immediately before a valid felt-heard message and remained visible after Adam advanced to Stage 2.

Earlier proposal `v01` covered canonical `<FeelHeardCheck>Y</FeelHeardCheck>` leakage. This run shows the model can also invent tool-style snake_case aliases, so the guard needs to treat those aliases as hidden control metadata too.

## Proposed Product Change

Extend the visible AI text scrubber to remove known canonical and snake_case control flag forms before persistence or rendering:

```text
FeelHeardCheck / feel_heard
ReadyShare / ready_share
NeedsReady / needs_ready
StrategyProposed / strategy_proposed
```

For Stage 1, parsing should preserve the gate signal when malformed visible aliases appear. For example:

```text
<thinking>Mode:Witness</thinking>
<feel_heard>Y</feel_heard>
Visible response.
```

should yield:

```text
offerFeelHeardCheck: true
response: "Visible response."
```

## Proposed Prompt Change

In the shared response protocol, explicitly forbid XML-style control tags, including both canonical and snake_case variants:

```text
Do not invent XML-style control tags such as <FeelHeardCheck>Y</FeelHeardCheck> or <feel_heard>Y</feel_heard>. Control flags are plain text lines inside <thinking> only.
```

## Regression Tests

Add focused backend tests:

- `cleanVisibleAIText("<feel_heard>Y</feel_heard> Visible")` returns `Visible`.
- `cleanVisibleAIText("ready_share: Y\n\nVisible")` returns `Visible`.
- `parseMicroTagResponse("<thinking>Mode:Witness</thinking><feel_heard>Y</feel_heard>Visible")` strips the tag and sets `offerFeelHeardCheck === true`.

## Acceptance Criteria

- Next Adam/Eve Stage 1 run does not render any `feel_heard`, `FeelHeardCheck`, `ready_share`, or `ReadyShare` marker in the DOM or saved message history.
- `mwf_handling` improves from `3` to at least `4`.
- `actor_fidelity` remains at least `4`.
- Hard invariant `internal_control_tag_exposed` passes.
