# MWF Prompt Proposal v05

Scenario: `adam-eve`
Source run: `eval/runs/20260505-193719-adam-eve-iter-01`
Focus: Keep optional context sharing from interrupting Stage 2 empathy completion.

## Evidence

The run scored `3.0` overall with `mwf_handling` at `3`. Actor fidelity passed, but the Stage 2 flow became structurally ambiguous after an optional context share:

- `transcripts/eve-stage2.md`: Eve received a `SHARE SUGGESTION` before completing and submitting her empathy statement for Adam.
- `scratch/2026-05-05-cmotg5kxk001rpx8n72h3psqv-eve.md`: Eve recorded that the optional context-share appeared during empathy work and later left her waiting on Adam with no Eve-side continuation.
- `transcripts/adam-stage2.md`: Adam received Eve's context and a share suggestion after his own submitted empathy statement, but the exchange did not reach a clean Stage 2 completion or validation gate in the run artifacts.

## Proposed Prompt Change

In `backend/src/services/stage-prompts.ts`, tighten Stage 2 context-share guidance so optional sharing is framed as a between-step gate rather than a mid-draft interruption:

```text
Only introduce optional partner-context sharing after the current user has completed the immediate empathy attempt or explicitly pauses because they cannot continue without naming their own experience. Do not let optional context sharing replace or interrupt the empathy attempt itself.

When optional context is shared, return the user to the exact Stage 2 task that was underway: either finishing their empathy attempt, revising after partner context, or reviewing the partner's completed attempt. Use concrete continuation language so neither partner is left waiting for an undefined next action.
```

## Product Guard To Pair With This Prompt

The product should make the active owner explicit after a context share:

```text
If shared context sets the guesser's empathy attempt to REFINING, the guesser must have an available continuation action unless they have already reached the configured evaluation stop stage. The subject should not remain blocked on a partner action that the UI does not expose.
```

## Regression Tests

- Add or extend a Stage 2 reconciler flow test where User B accepts an `OFFER_OPTIONAL` share while User B is still drafting empathy, then confirm User B can submit empathy and User A receives a concrete continuation.
- Re-run the Adam/Eve gold loop through Stage 2 and confirm the final statuses satisfy `stage_limit_reached_correctly`.

## Acceptance Criteria

- Optional context share no longer distracts from the active empathy attempt.
- After a context share, the next active side and action are unambiguous in both UI state and run artifacts.
- `mwf_handling` improves from `3` to at least `4`.
- `actor_fidelity` remains at least `4`.
