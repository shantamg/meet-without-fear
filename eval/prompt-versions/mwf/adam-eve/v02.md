# MWF Prompt Proposal v02

Scenario: `adam-eve`
Source run: `eval/runs/20260505-162548-adam-eve-iter-01`
Focus: Keep the Stage 1 feel-heard checkpoint from blocking continued disclosure.

## Evidence

The run scored `3.5` overall with `mwf_handling` at `3`. Adam's latest Stage 1 assistant turn said:

```text
What happens after she pulls away? Where does that leave you both?
```

At that moment the UI showed only `I feel heard`; the free-text input was gone. The scorer treated this as a handoff defect because Adam was forced to either advance or stop answering while MWF was still asking a substantive question.

This differs from the earlier passing Stage 1 smoke run (`20260505-161010-adam-eve-stage1-smoke-iter-01`, score `4`) where the short path did not expose this contradiction.

## Proposed Product Change

Make the Stage 1 feel-heard panel a checkpoint, not an input replacement. When `showFeelHeardConfirmation` is true in Stage 1, keep the chat input visible so the user can either:

- click `I feel heard`, or
- keep talking in response to the assistant.

This matches the existing Stage 1 prompt language that says the user can confirm below or keep talking.

## Proposed Prompt Change

Tighten `backend/src/services/stage-prompts.ts` so `FeelHeardCheck:Y` responses do not end with a fresh substantive question. The turn should close with a reflective confirmation/continuation invitation, for example:

```text
That's the deepest part of it: when the quiet is meant to protect you, it can end up confirming her fear instead. If that captures it, you can let me know below. If there's more, I'm still here.
```

Avoid this shape when `FeelHeardCheck:Y`:

```text
That's the deepest part of it. What happens after she pulls away?
```

## Regression Tests

Add or update `mobile/src/utils/__tests__/chatUIState.test.ts` so `aboveInputPanel === "feel-heard"` keeps `shouldHideInput === false`.

## Acceptance Criteria

- Next Adam/Eve Stage 1 run still offers `I feel heard` when the user appears heard.
- If the latest assistant message invites more disclosure, the text input remains available.
- Stage 1 `mwf_handling` improves from `3` to at least `4`.
- Actor fidelity remains at least `4`.
