Catherine named "a pattern of yelling and personal attacks during conflict." James received the topic "How we handle conflict when things get heated." The topic was private and neutral, but it lost the concrete behavioral signal the gold profile needed.

## Intended Behavior Change

Stage 0 topic framing should stay short and non-accusatory while preserving concrete behavioral signals that can be stated neutrally. It should not flatten words like "yelling" and "personal attacks" into generic language such as "conflict" or "communication."

## Proposed Prompt Addendum

Add to the Stage 0 topic draft constraints:

```text
Preserve the user's concrete behavioral signal when it can be stated neutrally. Do not flatten "yelling", "personal attacks", "threats", "stonewalling", or "drinking" into vague phrases like "conflict", "communication", or "things get heated".
```

Adjust the loaded-language example away from generic "when we're upset" phrasing:

```text
how we speak to each other when conflict escalates
```

## Regression Risk

The main risk is that neutral concrete terms could feel accusatory in lower-intensity scenarios. The prompt limits preservation to signals that can be stated neutrally and still forbids blame, contempt, names, and editorial interpretation.

## Rollback Notes

Remove the concrete-signal bullet from `buildInvitationPrompt` and restore the previous example if future runs show invitee defensiveness caused by over-specific topic wording.
