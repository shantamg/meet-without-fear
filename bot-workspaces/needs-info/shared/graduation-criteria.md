# Graduation Criteria

Rules for determining when an issue has enough information to leave the needs-info loop.

## Bug Reports — Graduate when ALL of:
- [ ] Identified the screen or feature affected
- [ ] Described the action the user took (steps to reproduce)
- [ ] Described expected vs actual behavior

## Feature Requests — Graduate when ALL of:
- [ ] Identified the user need or problem being solved
- [ ] Described the desired outcome (what "done" looks like)
- [ ] Identified who it affects (scope)

## Questions — Graduate when ANY of:
- [ ] Enough context to search docs or code for an answer
- [ ] The specific screen/feature/term in question is identified

## Auto-graduate triggers
- User provides a screenshot or video — often sufficient for bugs
- User references a specific error message — sufficient for bugs
- 3+ rounds of questions answered — assume enough context gathered

## Never graduate if:
- The category is still ambiguous (could be bug or feature)
- The user's responses contradict each other (ask for clarification)
- No concrete screen, feature, or workflow has been identified
