# GitHub Respond — Workspace Context

## Purpose

Handle GitHub PR interactions: respond to comments/mentions and perform code reviews.

## Stage Pointers

- `stages/respond/CONTEXT.md` — Reply to PR comments and mentions
- `stages/review/CONTEXT.md` — Perform thorough code review

## Shared Resources Used

- `shared/references/github-ops.md` — PR comment reply patterns, issue conventions

## Key Conventions

- Acknowledge immediately before doing heavy work
- For code change requests: checkout branch, fix, commit, push, reply
- If PR is not yours and you're just mentioned, don't make changes unless explicitly asked
